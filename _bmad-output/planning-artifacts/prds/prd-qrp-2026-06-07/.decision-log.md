## 2026-06-07 — New module: `macro` (central-bank / macroeconomic data)

- Add a distinct module **`macro`** for traditional macroeconomic / official
  public-sector data: central-bank policy rates (Fed funds), FOMC statements &
  minutes, balance-sheet data, monetary aggregates, economic projections, ECB/BoE
  policy rates, inflation & financial-stability reports.
- **`macro` ≠ `signal`.** macro = traditional macro/market/fundamental/official data.
  signal = alternative data (card transactions, satellite, web-scraping, geolocation,
  social sentiment, shipping/logistics, job postings).
- **Nuance recorded:** a dataset *derived* from central-bank sources (e.g. NLP
  sentiment of FOMC speeches, ML features from meeting transcripts) is **alternative
  data (`signal`)** even though its raw source is macro. Rule of thumb: raw official
  release → `macro`; derived/extracted feature → `signal`.
- macro is client-agnostic shared reference (like sym). Roadmap, not v1.
- Updated module family: `sym` · `macro` · `portfolios` · `signal` · `optima` · `analytics`.

## 2026-06-07 — Split old `optima` module → `backtest` + `optimiser`

- The former combined `optima` module is split into two: **`backtest`** (run a defined
  strategy/portfolio over history → track record) and **`optimiser`** (search an objective
  + constraints → optimal weights). Backtester name = **`backtest`**; the optimiser module
  is named **`optimiser`** (NOT `optima`).
- Relationship: `backtest` is foundational (the evaluation engine); `optimiser` depends on
  it (uses backtests to score candidates). Both consume sym (+ `signal`); both emit paper
  Portfolios that `analytics` measures.
- **Updated module family (8):** `sym` · `macro` · `altdata` · `signal` · `backtest` ·
  `optimiser` · `portfolios` · `analytics`.

## 2026-06-07 — Module taxonomy: split `signal` from alt-data

- The alt-data module is renamed **`altdata`** (display "Alternative Data") — it was
  previously (mis)named `signal`. Raw alt datasets: card transactions, satellite,
  web-scraping, geolocation, social sentiment, shipping/logistics, job postings.
- **`signal` is now a SEPARATE module** whose purpose is to *identify / produce signals*
  — derived features, factors, alpha signals — computed from the underlying data
  (sym + macro + altdata). The earlier "derived feature (e.g. NLP sentiment of FOMC
  speeches)" nuance now lands in `signal`, not in alt-data.
- `macro` unchanged.
- **Updated module family (7):** `sym` · `macro` · `altdata` · `signal` · `portfolios`
  · `optima` · `analytics`. (Module slug `altdata` for the two-word "alt data" name.)

## 2026-06-07 — Portfolios start with weights

- A portfolio's holdings are **weights to start** — an effective-dated vector of
  weights over sym_id constituents (summing to ~1), forming a time series. Analytics
  are computed from **weighted constituent returns** (sym's return data), not from
  share quantities. Share-quantity holdings and a transactions ledger (realized
  cash PnL) are a **later enhancement**, not the first cut.
